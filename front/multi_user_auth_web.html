✨ 自动化优化

系统现在会在认证成功后自动为您的项目启用以下必需的API服务：

  • Gemini Cloud Assist API
  • Gemini for Google Cloud API

说明：无需手动启用API，系统会自动处理这些配置步骤。

📁 高级选项：Google Cloud Project ID (不用管，直接点击获取链接即可) ▶
💡 提示：如果你不懂这是什么，可以留空此字段让系统自动检测项目ID

步骤一：认证链接

点击此链接进行 OAuth 认证

认证步骤：

  1. 点击上方认证链接，在新窗口中完成 Google 登录
  2. 授权应用访问您的 Google Cloud 项目
  3. 看到 "OAuth authentication successful!" 页面后，关闭该窗口
  4. 返回本页面，点击下方"获取认证文件"按钮

⚠️ 注意

当回源时，网页访问会显示报错。这时请手动将浏览器地址栏的 localhost 修改为 gcli-auth.sukaka.top，然后重新访问即可。

🚀 无法回源？试试快捷方式 ▼
📚 适用场景：
  • 云服务器、VPS等非本地环境
  • 防火墙阻止了回调端口访问
  • 网络环境无法正常回源
  • Docker容器端口映射问题
🔍 什么是回调URL？
完成Google OAuth授权后，浏览器地址栏显示的完整URL，通常看起来像这样：
http://localhost:8080/?state=abc123...&code=4/0AVMBsJ...&scope=email%20profile...
📋 使用步骤：
  1. 点击上方认证链接，完成Google授权
  2. 授权成功后，复制浏览器地址栏的完整URL
  3. 粘贴到下方输入框，点击获取凭证即可

步骤二：认证成功！
